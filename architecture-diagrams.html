2. Runtime And Data Flow

            
How a request moves through the app, including auth, Prisma, uploads, and AI flows.

          

          

        

        

          
sequenceDiagram
    autonumber
    actor User
    participant FE as Next.js Frontend
    participant MW as Middleware
    participant API as Route Handler
    participant Auth as NextAuth / Session
    participant Lib as Shared Domain Logic
    participant Prisma as Prisma ORM
    participant PG as PostgreSQL
    participant S3 as S3 Storage
    participant OR as OpenRouter

    User->>FE: Open page / interact with UI
    FE->>MW: Request protected route
    MW-->>FE: Allow or redirect by role

    FE->>API: Fetch JSON from /api/*
    API->>Auth: getServerSession()
    Auth-->>API: session + user role/id

    alt Project-scoped creator flow
        API->>Lib: ensureProjectAccess(projectId)
        Lib->>Prisma: validate role + membership
        Prisma->>PG: query project, members, pitches
        PG-->>Prisma: rows
        Prisma-->>Lib: result
        Lib-->>API: allow or deny
    end

    API->>Lib: business logic
    Lib->>Prisma: typed ORM queries
    Prisma->>PG: SQL via Prisma engine
    PG-->>Prisma: records / aggregates
    Prisma-->>Lib: domain data
    Lib-->>API: response model
    API-->>FE: JSON payload
    FE-->>User: updated UI

    alt Upload flow
        FE->>API: multipart/form-data upload
        API->>S3: PutObject
        S3-->>API: object stored
        API-->>FE: publicUrl + path
    end

    alt MODOC AI flow
        FE->>API: POST /api/modoc/chat
        API->>Auth: optional session lookup
        API->>Prisma: fetch context/history/project data
        Prisma->>PG: query data
        PG-->>Prisma: context records
        API->>OR: streamText(system prompt + messages)
        OR-->>API: streamed completion
        API-->>FE: streaming response
        FE-->>User: AI answer
    end
          

        

        

      


      

        

          

            
3. Security Architecture And Risks

            
Controls that exist today, plus the highest-risk exposure points visible in the repo.

    subgraph AUTHSEC[Authentication and Identity]
        NEXTAUTH[NextAuth + Prisma Adapter]:::good
        PROVIDERS[Providers:\nCredentials / Email / Google / GitHub]:::good
        JWT[JWT session strategy]:::good
        BCRYPT[bcryptjs password hashing]:::good
        SECRET[NEXTAUTH_SECRET env-based]:::good
        DEMO[Fallback demo password in auth.ts\nif passwordHash missing and env unset]:::risk
    end

    subgraph ACCESSSEC[Authorization]
        MWARE[Middleware role checks\n/admin /creator /music-creator /\ncompany routes]:::good
        SERVERCHECK[Server-side getServerSession checks]:::good
        PROJECTCHECK[Project membership validation\nmembers + pitches + admin role]:::good
        AUDIT[AdminAuditLog and ActivityLog]:::good
        DUPLICATION[Authorization logic partly duplicated\nacross many route files]:::med
    end

    subgraph APPSEC[Application Security]
        ORM[Prisma ORM\nreduced SQL injection risk]:::good
        INPUT[Basic input validation exists in many routes]:::med
        VALGAP[No formal validation library found\nno zod/yup/joi/valibot usage]:::med
        ERR[Some dev-mode error leakage in signup route]:::med
        HEADERS[No CSP / HSTS / X-Frame-Options config found]:::risk
        RATELIMIT[No application-level rate limiting found]:::risk
    end

    subgraph DATASEC[Data and Privacy]
        ENV[Secrets templated in .env.example\nnot hardcoded in repo]:::good
        EMAILLEAK[Public listing endpoints expose user email\nin crew/casting directory responses]:::risk
        AIPERSIST[AI conversations persisted in DB]:::note
        AUDITDATA[Audit trails for some privileged actions]:::good
    end

    subgraph ENDPOINTRISKS[High-Risk Endpoint Findings]
        UPLOADRISK[Upload route allows file upload\nwithout visible auth guard\n/api/upload/content-media]:::risk
        MODOCRISK[MODOC chat can run with null userContext\nwhen API key exists\nanonymous cost/abuse exposure]:::risk
        SIGNUP[Signup has basic checks only\nno rate limiting / abuse controls]:::risk
    end

    subgraph OPSSEC[Operational Security]
        TESTS[Only lint + typecheck + smoke test]:::med
        NOTESTS[No formal unit/integration/e2e test framework found]:::risk
        NOCI[No CI workflow found]:::risk
        NODOCKER[No container/deploy config found in repo]:::med
        LOGGING[Prisma error logging configured;\nmore verbose in development]:::note
    end
